JSP: JavaServer Pages

Teoría:

JavaServer Pages (JSP) es una tecnología de Java que permite a los desarrolladores crear páginas web dinámicas basadas en HTML, XML u otros formatos, integrando código Java para generar contenido dinámico. JSP simplifica el desarrollo de aplicaciones web al combinar la lógica de negocio escrita en Java con la presentación en páginas web, actuando como una capa de vista en aplicaciones basadas en el modelo MVC (Modelo-Vista-Controlador).

JSP no es un framework independiente, sino una extensión de la tecnología de Servlets. Las páginas JSP son traducidas a servlets por el contenedor web (como Apache Tomcat) en tiempo de compilación, lo que permite ejecutar lógica Java en el servidor antes de enviar el resultado al cliente como una página HTML. Esto elimina la necesidad de escribir código HTML directamente en servlets, facilitando la creación y mantenimiento de interfaces web dinámicas.

Conceptos Fundamentales

  • Página JSP:Un archivo con extensión .jsp que contiene una mezcla de HTML (o XML) y elementos JSP como directivas,expresiones, scriptlets y etiquetas personalizadas. Este archivo es procesado por el servidor para generar contenido dinámico.
  • Directivas JSP:Instrucciones que controlan cómo se procesa la página JSP. La directiva más común es <%@ page %>, que define configuraciones como el tipo de contenido o la importación de clases Java.
  • Expresiones y Scriptlets:Las expresiones (<%= %>) permiten mostrar valores dinámicos en la página, mientras que los scriptlets (<% %>) contienen código Java para lógica de negocio embebida en la página.
  • JSTL (JavaServer Pages Standard Tag Library): Una biblioteca de etiquetas estándar que simplifica el desarrollo de JSP, reduciendo la necesidad de scriptlets y mejorando la legibilidad del código.

Práctica:

Para utilizar JSP, es necesario configurar un proyecto web en un servidor compatible (como Tomcat) y crear archivos JSP que combinen HTML con elementos dinámicos. A continuación, se describen los pasos y componentes principales para trabajar con JSP.

Creación de una Página JSP:

Una página JSP es un archivo con extensión .jsp que mezcla contenido estático (HTML) con elementos dinámicos generados por código Java. La estructura básica incluye directivas, scriptlets, expresiones y, opcionalmente, etiquetas JSTL.

Elementos clave de JSP:

  • <%@ page %>: Directiva que configura la página, como el lenguaje, el tipo de contenido o las importaciones de clases.
  • <%= variable %>: Expresión que inserta el valor de una variable o expresión Java directamente en la salida HTML.
  • <% código Java %>: Scriptlet que contiene lógica Java para procesar datos o controlar el flujo de la página.
  • <%@ taglib %>: Directiva para importar bibliotecas de etiquetas personalizadas, como JSTL, que permiten escribir JSP sin scriptlets.

Configuración del Proyecto

Para usar JSP, el proyecto debe incluir un servidor web compatible (como Tomcat) y una estructura de directorios estándar para aplicaciones web Java. El archivo web.xml en la carpeta WEB-INF configura los servlets y mapeos, mientras que las páginas JSP suelen colocarse en el directorio raíz o en subdirectorios específicos.

Pasos principales:
  • Crear un proyecto web dinámico en un IDE como Eclipse o IntelliJ.
  • Configurar un servidor web (Tomcat, Jetty, etc.) en el proyecto.
  • Crear archivos JSP con contenido estático y dinámico.
  • Desplegar la aplicación en el servidor para probar las páginas.

Ejemplos de Uso

Se presenta un ejemplo de cómo crear una página JPS que genere una lista de productos directamente en la pagina.

listaProductos.jsp:
Productos Disponibles

<%@ page language="java" contentType="text/html; charset=UTF-8" pageEncoding="UTF-8"%>
<%@ page import="java.util.ArrayList" %>
<%@ page import="java.util.List" %>
<%
class Producto {
    private Long id;
    private String nombre;
    private double precio;

    public Producto(Long id, String nombre, double precio) {
        this.id = id;
        this.nombre = nombre;
        this.precio = precio;
        }

        public Long getId() { return id; }
        public String getNombre() { return nombre; }
        public double getPrecio() { return precio; }
        }
List productos = new ArrayList<>();
        productos.add(new Producto(1L, "Teclado Mecánico", 99.99));
        productos.add(new Producto(2L, "Ratón Óptico", 49.99));
        productos.add(new Producto(3L, "Monitor LED", 199.99));
%>
Resultado Final:

Al acceder a la página JSP listaProductos.jsp en un servidor como Tomcat, el scriptlet crea una lista de productos y el bucle genera una tabla HTML con los datos.
