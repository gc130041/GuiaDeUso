Maven y pom.xml

¿Qué es Maven?

Maven es una herramienta de gestión y automatización de proyectos para aplicaciones Java. Simplifica la construcción, reporte y documentación de proyectos a través de un modelo de objetos estandarizado y un archivo de configuración centralizado llamado pom.xml.

Ventajas de usar Maven

  • Gestión automática de dependencias y versiones.
  • Estandarización en la estructura de proyectos.
  • Integración con sistemas de construcción y CI/CD.
  • Facilita la gestión de ciclos de vida del proyecto (compilar, testear, empaquetar, desplegar).

¿Qué es el archivo pom.xml?

El archivo pom.xml (Project Object Model) es el archivo de configuración principal que Maven utiliza para administrar el proyecto. Define dependencias, plugins, información del proyecto y configuraciones del ciclo de vida.

Estructura básica del pom.xml

<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0
         http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.0.0</modelVersion>

    <groupId>com.ejemplo</groupId>
    <artifactId>mi-proyecto</artifactId>
    <version>1.0.0</version>

    <dependencies>
        <dependency>
            <groupId>junit</groupId>
            <artifactId>junit</artifactId>
            <version>4.12</version>
            <scope>test</scope>
        </dependency>
    </dependencies>
</project>

Comandos básicos de Maven

  • mvn clean: Limpia el directorio de compilación.
  • mvn compile: Compila el código fuente del proyecto.
  • mvn test: Ejecuta los tests unitarios.
  • mvn package: Empaqueta el proyecto en un archivo JAR o WAR.
  • mvn install: Instala el artefacto en el repositorio local.
  • mvn deploy: Despliega el artefacto a un repositorio remoto.

Repositorio local de Maven

Maven guarda las dependencias descargadas y los artefactos construidos en un repositorio local, ubicado normalmente en ~/.m2/repository. Esto evita descargar repetidamente las mismas dependencias en diferentes proyectos.
